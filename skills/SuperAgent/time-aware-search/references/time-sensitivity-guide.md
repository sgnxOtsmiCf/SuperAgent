# 时间敏感性判断指南

## 为什么需要时间感知技能？

大语言模型（LLM）的训练数据有明确的截止时间，这导致模型在回答时效性问题时存在以下问题：

1. **时间认知偏差**：模型不知道"现在"是什么时候
2. **信息过时**：基于训练数据截止前的信息回答
3. **相对时间误解**：无法理解"最近"、"今年"等相对时间词汇

## 时间敏感性等级划分

### 高敏感度（必须使用时间感知技能）

涉及以下领域的问题：
- **新闻事件**：时事、热点、突发事件
- **金融市场**：股价、汇率、行情
- **政策法规**：法律、政策、规定
- **科技动态**：新产品发布、技术突破
- **天气信息**：当日/近期天气预报
- **活动信息**：会议、展览、演出

关键词：
- 今天、现在、当前、最新、最近
- 新闻、动态、进展、行情、趋势
- 2026年、本月、本周

### 中敏感度（建议使用时间感知技能）

涉及以下领域的问题：
- **行业报告**：市场分析、行业研究
- **统计数据**：人口、经济、社会数据
- **产品信息**：价格、规格、可用性

关键词：
- 近年来、近几年、近期
- 报告、数据、统计

### 低敏感度（可不使用时间感知技能）

涉及以下领域的问题：
- **概念解释**：定义、原理、方法
- **历史事实**：已发生的历史事件
- **通用知识**：科学、文化、常识

关键词：
- 是什么、为什么、如何
- 历史、传统、经典

## 工具调用最佳实践

### 1. 时间工具（DateTimeTool）

**调用时机**：
- 检测到任何时间敏感词后立即调用
- 不要缓存时间超过5分钟

**参数**：
- 无需参数，获取当前系统时间

**返回值处理**：
```javascript
{
  datetime: "2026-04-16T14:30:00+08:00",
  date: "2026-04-16",
  time: "14:30:00",
  timezone: "Asia/Shanghai",
  weekday: "Thursday"
}
```

### 2. 搜索工具（WebSearchTool / TavilySearchTool）

**调用时机**：
- 获取当前时间后
- 构建带时间限定的查询

**查询构建原则**：
1. 保留用户原始意图
2. 添加时间限定词
3. 使用具体时间而非相对时间

**示例**：
- 用户："最近有什么新闻"
- 时间：2026-04-16
- 查询："新闻 2026年4月 最新"

### 3. 结果筛选

优先选择：
1. 时间戳在目标范围内的结果
2. 权威来源（政府、官方媒体）
3. 高相关度结果

## 错误处理

### 时间获取失败

**现象**：DateTimeTool 返回错误或超时

**处理**：
1. 重试最多3次
2. 如仍失败，使用系统提示中的时间（如有）
3. 向用户说明："时间服务暂时不可用，以下信息基于..."

### 搜索无结果

**现象**：在指定时间范围内未找到相关信息

**处理**：
1. 扩大时间范围（如从7天扩大到30天）
2. 调整搜索关键词
3. 向用户说明并询问是否扩大搜索范围

### 结果冲突

**现象**：多个来源提供相互矛盾的信息

**处理**：
1. 优先选择权威来源
2. 优先选择时间戳更新的结果
3. 向用户呈现多方观点并标注来源
